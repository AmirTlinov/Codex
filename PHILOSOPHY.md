# Philosophy

Codex CLI and codex-rs favor safe, deterministic automation with explicit boundaries.

- Safety before autonomy: sandbox + approvals are the default control surface.
- Deterministic behavior: context assembly, tool execution, and config resolution are inspectable and repeatable.
- Small, composable modules: prefer narrow interfaces and feature flags over sprawling surfaces.
- Evidence before action: read and verify before changes.
- Cross-platform reliability: keep behavior consistent across supported OSes.
